---
title: 追踪器配置
---

The Umami tracker provides several properties that allow you to configure its behavior.

### data-host-url

默认情况下，Umami 将数据发送到脚本所在的位置。你可以重写此属性，将数据发送到其他位置。

用法：

```html
<script
  defer
  src="http://mywebsite.com/umami.js"
  data-website-id="94db1cb1-74f4-4a40-ad6c-962362670409"
  data-host-url="http://stats.mywebsite.com"
></script>
```

### data-auto-track

默认情况下，Umami 会自动跟踪所有页面访问和事件。你可以禁用此行为，并使用[追踪器函数](/docs/tracker-functions)自行跟踪事件。

用法：

```html
<script
  defer
  src="http://mywebsite.com/umami.js"
  data-website-id="94db1cb1-74f4-4a40-ad6c-962362670409"
  data-auto-track="false"
></script>
```

### data-domains

如果你希望追踪器仅在特定域上运行，可以将它们添加到你的追踪器脚本中。这是一个以逗号分隔的域名列表。适用于在开发/测试环境中工作时有用。

用法：

```html
<script
  defer
  src="http://mywebsite.com/umami.js"
  data-website-id="94db1cb1-74f4-4a40-ad6c-962362670409"
  data-domains="mywebsite.com,mywebsite2.com"
></script>
```

### Google Tag Manager

Google Tag Manager 将剥离追踪器的属性，你可以通过以下格式来绕过此问题。

用法：

```html
<script>
  (function () {
    var el = document.createElement('script');
    el.setAttribute('src', 'http://mywebsite.com/umami.js');
    el.setAttribute('data-website-id', '94db1cb1-74f4-4a40-ad6c-962362670409');
    document.body.appendChild(el);
  })();
</script>
```
