---
title: 旅程报告
---

Umami Journey 提供数据分析，以优化网站性能和增强用户体验。用户旅程描绘出访客为实现目标（例如购买或注册新闻通讯）所采取的步骤。通过分析这些路径，您可以识别并消除痛点，最终提高客户满意度。

该报告通过展示您的网站的主要用户旅程来运作。您可以逐步浏览每个步骤，探索用户从哪里开始、在哪里中断，以及最后到达哪里。

## 参数

- `网站`:（必填）您想要运行报告的网站。
- `日期范围`:（必填）报告的日期范围。
- `步骤`:（必填）在用户旅程中使用的顺序步骤条件。数量范围为 3 到 7 步。
- `起始步骤`: 用户旅程的第一个步骤。可以是 URL 或事件。
- `结束步骤`: 用户旅程的最后一个步骤。可以是 URL 或事件。

## 创建报告

### 第一步：选择网站

![image](/images/docs/v2/journey-website.png)

### 第二步：选择日期范围

![image](/images/docs/v2/journey-date-range.png)

### 第三步：选择步骤

选择步骤的数量和可选的步骤条件，例如您的用户从哪里开始和/或结束他们的旅程。

![image](/images/docs/v2/journey-steps.png)

### 第四步：运行查询并保存

点击 **运行查询** 查看结果。为了保存报告，请添加标题和描述，然后点击 **保存**。报告及其参数将被保存以供将来使用。

![image](/images/docs/v2/journey-details.png)

### 第五步：探索旅程

选择特定的 URL 或事件，以查看有多少用户到达该步骤。

![image](/images/docs/v2/journey-select-node.png)

悬停在特定的 URL 或事件上，以探索其他相关路径。

![image](/images/docs/v2/journey-hover-node.png)

报告将显示每个步骤的总访客流失百分比。

![image](/images/docs/v2/journey-drop-off.png)
