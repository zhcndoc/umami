# API 密钥

为了使用 Umami Cloud 访问你的数据，你需要生成一个 API 密钥。

在参考部分中，除了列在“不允许”部分的一些路由之外，Umami Cloud 中的所有 API 路由都可以使用 API 密钥进行访问。

## 生成你的密钥

要在 Umami Cloud 中创建一个密钥，请转到 `Accounts` > `API Keys` 并点击 `Create key` 按钮。

<img src="/images/cloud/create-api-key.png" />

通过点击文本输入框中的 `visible` 图标，你可以显示你的密钥。

## 使用你的密钥

你可以通过查询以下地址来访问你的 Umami Cloud 数据。

`https://api.umami.is/v1`

只需通过一个名为 `x-umami-api-key` 的自定义头部传入你的 API 密钥。

例如，使用 `curl`，你的请求将如下所示：

```shell
curl https://api.umami.is/v1/websites
   -H "Accept: application/json"
   -H "x-umami-api-key: <api-key>"
```

## 使用 API 客户端

设置以下必需变量以便 API 客户端使用你的 API 密钥。

```dotenv
UMAMI_API_KEY={Your API key}
UMAMI_API_CLIENT_ENDPOINT=https://api.umami.is/v1
```

有关使用 API 客户端的更多详细信息，请参阅 [API client](/docs/api/api-client)。

## 限制

每个 API 密钥在15秒内限制为50次调用。

## 不允许

以下 API 路由不允许使用你的 API 密钥。

```text
/me/password
/users
/users/*
```

## 删除你的密钥

你可以在 `Accounts` > `API Keys` 下删除你的密钥，单击要删除的特定密钥的 `Delete` 按钮。

一旦删除 API 密钥，将无法恢复。
