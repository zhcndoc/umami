---
title: 托管
---

有很多方法可以让 Umami 运行起来。您只需要一个数据库（可以是 [MySQL](https://www.mysql.com/) 或 [PostgreSQL](https://www.postgresql.org/)）和一个可以运行 [Node.js](https://nodejs.org/)（18.18 或更新版本）的服务器。

## 单服务器

在这种设置中，数据库和 Web 服务器在同一台机器上运行。您可以选择在专用 Web 服务器（如 [Nginx](https://www.nginx.com/) 或 [Apache](https://httpd.apache.org/)）后面运行 Umami，并将请求代理到 Umami。

您可以查看 [在 DigitalOcean 上运行](/docs/guides/running-on-digitalocean) 指南，以了解如何设置服务器。

## 分离的数据库和 Web 服务器

在这种设置中，数据库在独立的机器上运行，并允许远程连接。您的 Web 服务器在另一台安装了 Umami 的机器上运行，并通过远程连接到数据库以存储和检索数据。

## 应用托管

在这个设置中，您将需要一个应用托管提供商为您托管 Umami 应用。您仍然需要一个在其他地方运行的数据库，以便应用能够连接。以下是几个可以托管 Umami 的应用托管提供商。Vercel、Netlify 和 Render 提供免费层级服务。

- [Vercel](https://vercel.com/)
- [Netlify](https://www.netlify.com/)
- [Render](https://render.com/)
- [Railway](https://railway.app/)

在部署时，构建命令为 `npm run build`，发布目录为 `.next`。

## 托管数据库

托管数据库是指托管方为您提供一个正在运行的数据库，您无需安装或维护。以下是一些托管数据库提供商。

- [DigitalOcean](https://www.digitalocean.com/products/managed-databases/)
- [Linode](https://www.linode.com/products/databases/)

- [Google Cloud SQL](https://cloud.google.com/sql/)
- [Microsoft Azure](https://azure.microsoft.com/en-us/services/#databases)
