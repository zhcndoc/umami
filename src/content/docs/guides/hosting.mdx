title: 托管
---

There are many ways you can get Umami up and running.
All you need is a database (either [MySQL](https://www.mysql.com/) or [PostgreSQL](https://www.postgresql.org/))
and server that can run [Node.js](https://nodejs.org/) (18.17 or newer).

## 单服务器

在这种设置下，你可以在同一台机器上运行数据库和 Web 服务器。你还可以选择在 Umami 后面运行一个专用的 Web 服务器，如 [Nginx](https://www.nginx.com/)或 [Apache](https://httpd.apache.org/)，并将请求代理到 Umami。

你可以查看[在 DigitalOcean 上运行](/docs/running-on-digitalocean)指南，了解如何设置服务器。

## 分离的数据库和 Web 服务器

在这种设置下，你的数据库在自己的机器上运行，允许远程连接。你的 Web 服务器在另一台安装了 Umami 的机器上运行，并远程连接到数据库以存储和检索数据。

## 应用托管

在这种设置下，你将让应用托管提供商为你托管 Umami 应用。你仍然需要在其他地方运行一个数据库，应用可以连接到该数据库。以下是一些可以托管 Umami 的应用托管提供商。Vercel、Netlify 和 Railway 都有免费的套餐可供选择。

- [Vercel](https://vercel.com/)
- [Netlify](https://www.netlify.com/)
- [Render](https://render.com/)
- [Railway](https://railway.app/)

对于部署，构建命令是 `npm run build`，发布目录是 `.next`。

## 托管数据库

托管数据库是指主机提供运行中的数据库，无需你安装或维护。以下是一些托管数据库的提供商。

- [DigitalOcean](https://www.digitalocean.com/products/managed-databases/)
- [Scaleway（欧洲）](https://www.scaleway.com/en/database/)
- [Amazon Web Services](https://aws.amazon.com/products/databases/)
- [Google Cloud SQL](https://cloud.google.com/sql/)
- [Microsoft Azure](https://azure.microsoft.com/en-us/services/#databases)
- [Render](https://render.com/pricing#databases)
- [Heroku](https://www.heroku.com/postgres)
- [Railway](https://docs.railway.app/databases/postgresql/)
- [Supabase](https://supabase.com/database)
- [PlanetScale](https://planetscale.com/)
