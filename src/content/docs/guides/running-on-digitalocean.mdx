# 在 DigitalOcean 上运行

[DigitalOcean](https://m.do.co/c/c9ebc1c0928d) 是一个经济实惠的云主机提供商，可以让你托管你自己的 Umami 设置。在这个设置指南中，我们将安装 [Ubuntu](https://ubuntu.com/)、[PostgreSQL](https://www.postgresql.org/) 或者 [MySQL](https://www.mysql.com/) 数据库、[Nginx](https://www.nginx.com/) web 服务器、[Node.js](https://nodejs.org/) 和 Umami。DigitalOcean 还有一个 [NodeJS droplet build](https://marketplace.digitalocean.com/apps/nodejs)，其中包含了 Node.js、Ubuntu 和 Nginx，可以让你更快地开始。

对于个人使用，你可以从每月 $5 起步，并根据需要进行扩展。你可以使用这个**[链接](https://m.do.co/c/c9ebc1c0928d)** 在前60天获得100美元的信用。

注意，这些步骤可以在任何提供 Ubuntu 的云主机提供商上重复。

## 运行 Umami

你可以简单地运行 `npm start` 来启动 Umami，但强烈推荐使用像 [PM2](https://pm2.keymetrics.io/) 这样的进程管理器，它可以为你处理重启。

使用 PM2 运行：

```shell
npm install pm2 -g
cd umami
pm2 start npm --name umami -- start
pm2 save
```

## 使用 Nginx 进行代理

现在 Umami 已经在运行，你可以使用 Nginx 将域名或子域名的请求代理到 Umami。

以下配置将把 `umami.yourdomain.com` 的所有请求转发到本地的 Umami 实例。

```nginx
server {
  server_name umami.yourdomain.com;

  location / {
    proxy_pass http://localhost:3000;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header Host $host;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  }
}
```

以下配置将允许你在你的域名的子路径上托管 Umami（例如：website.com/stats）。这需要在你的 `.env` 文件中设置环境变量 `BASE_PATH=/stats`。

```nginx
server {
  ...
  location /stats/_next/static/ {
    alias /your_install_location/umami/.next/static/;
    access_log off;
    expires max;
  }
  location /stats {
    proxy_pass http://127.0.0.1:3000;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header Host $host;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  }
  ...
}
```
