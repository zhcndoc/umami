> [Heroku](https://www.heroku.com/) is a container-based cloud Platform as a Service (PaaS). Developers use Heroku to deploy, manage, and scale modern apps. The platform is elegant, flexible, and easy to use, offering developers the simplest path to getting their apps to market.
>
> From the ["What is Heroku?" section](https://www.heroku.com/about) of their website.

Heroku 提供了一个免费的托管服务，可以与 Umami 以及通过附加组件使用的数据库一起使用。在进行此设置指南之前，你不需要设置数据库。

## 设置

[![部署](https://www.herokucdn.com/deploy/button.svg)](https://heroku.com/deploy?template=https://github.com/umami-software/umami)

_使用上面的按钮自动化所有步骤_

### 网站

1. 派生[Umami 仓库](https://github.com/umami-software/umami)的副本。
2. 在[Heroku](https://heroku.com/)上创建一个帐户。
3. 从仪表盘页面，点击 **New > Create new app**。
4. 选择一个 **App 名称**，然后点击 **Create app**。
5. 在 **Deployment method** 下点击 **GitHub**，按照说明将 Heroku 连接到 GitHub。
6. 搜索仓库并点击 **Connect**。

### 数据库

1. 导航到 **Resources** 标签页，点击 **Find more add-ons** 按钮。
2. 搜索 **Heroku Postgres**，并按照其说明安装附加组件。
3. 附加组件将自动设置 `DATABASE_URL`；你不需要手动设置它。
4. 你将需要通过按照[安装](/docs/install)文档中的**创建数据库表**部分的说明来设置数据库表。
   你可以通过按照 **Resources > Heroku Postgres > Settings > Database Credentials** 的路径找到临时连接详情。
   要执行文档中找到的命令，请点击 **More > Run Console**。

### 构建和部署

1. 在设置环境变量和设置数据库后，点击 **Deploy > Manual Deploy > Deploy Branch** 按钮。
2. 构建完成后，网站应该可以访问。请按照仪表盘顶部的 **Open app** 按钮进行查看。
3. 请从 [登录](/docs/login) 步骤开始，按照 **入门指南** 进行操作。
