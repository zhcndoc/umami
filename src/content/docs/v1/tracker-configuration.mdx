---
title: 跟踪器配置
---

Umami 跟踪器提供了多个属性，允许您配置其行为。

### data-host-url

默认情况下，Umami 将数据发送到脚本所在的位置。您可以覆盖此设置，将数据发送到另一个位置。

用法：

```html
<script
  defer
  src="http://mywebsite.com/umami.js"
  data-website-id="94db1cb1-74f4-4a40-ad6c-962362670409"
  data-host-url="http://stats.mywebsite.com"
></script>
```

### data-auto-track

默认情况下，Umami 会自动跟踪所有页面浏览和事件。您可以禁用此行为，并使用 [跟踪器函数](/docs/tracker-functions) 自行跟踪事件。

用法：

```html
<script
  defer
  src="http://mywebsite.com/umami.js"
  data-website-id="94db1cb1-74f4-4a40-ad6c-962362670409"
  data-auto-track="false"
></script>
```

### data-do-not-track

您可以配置 Umami 尊重访问者的 **请勿追踪** 设置。

用法：

```html
<script
  defer
  src="http://mywebsite.com/umami.js"
  data-website-id="94db1cb1-74f4-4a40-ad6c-962362670409"
  data-do-not-track="true"
></script>
```

### data-cache

如果您从同一用户那里获得大量页面浏览，例如在论坛网站上，您可以缓存一些数据以提高跟踪脚本的性能。

_注意：这将使用会话存储，因此您可能需要通知用户。_

用法：

```html
<script
  defer
  src="http://mywebsite.com/umami.js"
  data-website-id="94db1cb1-74f4-4a40-ad6c-962362670409"
  data-cache="true"
></script>
```

### data-domains

如果您希望跟踪器仅在特定域上运行，您可以将它们添加到您的跟踪器脚本中。这是一个以逗号分隔的域名列表。如果您在预发布/开发环境中工作，这将很有帮助。

用法：

```html
<script
  defer
  src="http://mywebsite.com/umami.js"
  data-website-id="94db1cb1-74f4-4a40-ad6c-962362670409"
  data-domains="mywebsite.com,mywebsite2.com"
></script>
```

### Google Tag Manager

Google Tag Manager 会去掉跟踪器的属性，您可以使用以下格式绕过此问题。

用法：

```html
<script>
  (function () {
    var el = document.createElement('script');
    el.setAttribute('src', 'http://mywebsite.com/umami.js');
    el.setAttribute('data-website-id', '94db1cb1-74f4-4a40-ad6c-962362670409');
    document.body.appendChild(el);
  })();
</script>
```
