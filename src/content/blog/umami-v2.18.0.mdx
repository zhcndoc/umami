在本次 Umami 版本中，我们引入了新的归因报告、唯一 ID、新的追踪器属性等多项新功能！

# 功能

## 归因报告

新的归因报告允许你追踪用户的来源渠道，帮助你衡量广告活动的效果并优化广告投入。该报告还集成了收入数据，让你可以看到哪些渠道为销售额贡献最大。

阅读[文档](https://umami.is/docs/reports/report-attribution)。

## 唯一 ID

我们为会话引入了一个新的属性——**唯一 ID（distinct ID）**。这是一个新参数，允许你为每个会话分配一个 ID，便于后续识别用户。

要设置唯一 ID，可以使用追踪器的 `umami.identify` 方法：

```js
umami.identify('bob@aol.com');
```

在服务器环境中，你可以通过发送带有 `id` 参数的事件负载来实现：

```json
{
  "payload": {
    "hostname": "your-hostname",
    "language": "en-US",
    "referrer": "",
    "screen": "1920x1080",
    "title": "dashboard",
    "url": "/",
    "website": "your-website-id",
    "id": "bob@aol.com"
  },
  "type": "event"
}
```

随后，你可以在会话页面搜索该唯一 ID。

![image](https://github.com/user-attachments/assets/c4e6f6bc-2d61-40cb-bea2-31bce8204872)

阅读[文档](https://umami.is/docs/distinct-ids)。

## data-before-send 属性

我们引入了一个新的追踪器属性 **`data-before-send`**，允许你定义一个函数，在数据发送前被调用。你可以检查并修改发送的数据负载，或完全取消发送。该函数接受两个参数，`type` 和 `payload`。若要继续发送，函数需返回一个负载对象；如果要取消发送，则返回一个假值。

首先，定义该回调函数：

```js
function beforeSendHandler(type, payload) {
  if (checkPayload(payload)) {
    return payload;
  }
  return false;
}
```

然后通过 `data-before-send` 属性传入函数名：

```html
<script
  defer
  src="/script.js"
  data-website-id="7801d46c-afd2-424c-8680-4ea713b2002a"
  data-before-send="beforeSendHandler"
></script>
```

## 图标的备用主机

默认用于显示网站网站图标 (favicon) 的主机是 `icons.duckduckgo.com`。现在你可以通过环境变量 `FAVICON_URL` 使用备用主机。你需要传入一个包含可替换域名标记的字符串。例如：

`https://icons.duckduckgo.com/ip3/{{domain}}.ico`

你可以使用的替代方案有：

`https://www.google.com/s2/favicons?domain={{domain}}`  
`https://logo.clearbit.com/{{domain}}`

# 迁移

本版本包含一些数据库迁移，新增了用于处理 UTM 参数的新列。迁移将在构建过程中自动执行。

这些迁移仅应用于新架构，不会迁移旧数据。原因是为避免对拥有大量数据的用户造成构建过程阻塞。相反，我们提供了脚本，你可以单独运行这些脚本来进行数据迁移。

想要迁移本版本数据，请使用此处的对应数据库 SQL 脚本：[转换 UTM & clid 列](https://raw.githubusercontent.com/umami-software/umami/master/scripts/data-migrations/convert-utm-clid-columns.sql)
