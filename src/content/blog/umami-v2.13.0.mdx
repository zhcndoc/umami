Umami v2.13.0 已经发布！此次更新引入了两个新的界面：事件和会话，以及保存会话数据、会话档案、错误修复、语言更新等更多功能。以下是所有详细信息：

# 新的事件界面

事件界面快速总结了您定义时间范围内记录的所有自定义事件。显然，使用自定义事件是此界面的必要条件。以下是新事件界面的一些关键特性：

- **集中事件信息**：我们将事件图表和指标表从概览界面移至此专用空间，使您更容易关注事件特定数据。
- **活动流**：向下滚动事件界面（在同一界面内）将显示所有您的事件活动的表格，包含访客头像。这种视觉表示帮助您快速识别谁（虽然是匿名的）正在执行特定事件。
- **简化的自定义数据访问**：旧的事件数据界面已经弃用。现在，您可以在事件界面的“属性”标签下访问所有自定义数据。此部分提供所有自定义数据属性及其值的详细概述。
- **增强的可见性**：通过这种新布局，您可以更轻松地跟踪和分析自定义事件的表现，从而做出更明智的决策。

# 新的会话界面

新的会话界面总结了会话数据。首先，提供了按天和小时的每周会话数据细分。视觉辅助工具（蓝色圆圈）帮助您查看哪个天/小时会话数量较高（由较大的蓝色圆圈表示）或较低（由较小的蓝色圆圈表示）。
以下是新会话界面的一些关键特性：
每小时流量细分：显示一周内的每小时流量细分。此功能让您识别网站或应用的高峰时段，帮助您优化内容传递和服务器资源。

![image](/images/blog/screenshot-sessions-screen.png)

## 访客档案

点击活动流中任意头像会弹出新页面，显示该特定访客的详细信息，包括他们的活动历史记录。

![image](/images/blog/screenshot-visitor-profile.png)

# 保存会话数据

我们引入了保存会话数据的支持。此功能与自定义事件数据类似，开启了用户追踪和分析的新可能：
简单实施：使用跟踪脚本中的 identify 函数来保存当前会话的数据。保存的会话数据将出现在属性下的新会话档案页面中。

例如：

```javascript
umami.identify({ email: 'bob@aol.com' });
```

# 错误修复和改进

根据您的反馈，我们不断努力改善 Umami。在此次发布中，我们解决了多个问题：

- 修复了被忽略的 PORT 变量
- 修复了搜索框消失的问题
- 将 NextJS 升级至 v14.2.5
- 将 Prisma 升级至 v5.17.0
- 为团队经理添加了将网站转移到团队的能力

# 语言更新

我们致力于让 Umami 适用于全球用户。此次发布包括以下语言更新：

- 加泰罗尼亚语（西班牙）
- 韩语
- 波兰语
- 中文
- 德语
- 高棉语
- 白俄罗斯语
- 日语
