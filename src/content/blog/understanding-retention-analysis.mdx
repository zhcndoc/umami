理解用户行为不仅仅在于用户如何发现和开始使用你的产品。同样重要的是他们在一段时间内如何继续与产品互动。这就是留存分析发挥作用的地方。Umami 留存报告可以作为这个过程的一部分，突出短期和长期用户参与，使你能够将其与营销活动、产品公告、功能发布、社交媒体帖子等匹配，以查看哪些因素影响留存率。

Umami 留存报告是产品分析报告套件中的一部分，允许团队可视化和分析用户留存模式。它展示了用户多频繁地返回你的网站或产品，帮助你理解短期和长期的参与趋势，并识别改善结果的机会。

## 留存报告主要特点

在我们跳入具体用例之前，让我们看看留存报告的一些主要特点：

1. **群体分析**：该报告使用群体分析按用户访问你网站的时间对用户进行分组。这让你可以跟踪不同用户组的参与变化。例如，如果你在6月15日推出了一项新功能，你可以在接下来的几天内跟踪那一群体的用户，以了解新功能对留存率的影响。

2. **每日访客跟踪**：留存报告基于用户的每日群体。

3. **回访见解**：你可以看到这些用户在首次访问后的特定天数内返回的频率，显示你的产品的“黏性”。

4. **灵活的时间框架**：你可以分析特定月份和年份的留存情况，以比较不同时间段或季节的留存率。

5. **可视化展示**：数据以群体图的形式呈现，使你能够快速识别趋势和模式。

有关如何设置留存报告的详细说明，请参考我们的 [文档](https://umami.is/docs/reports/report-retention)。

## 留存报告用例

### 用例示例：分析入职改进的影响

产品团队通常会投入大量资源来改善入职过程，目的是提高用户参与和留存率。留存报告可以帮助你衡量这些努力的有效性。

#### 设置

1. 确定你实施入职变更的日期。

2. 创建变更前和变更后的月份的留存报告。

3. 比较这两个时间段的留存模式。

#### 分析方法

设置好这些报告后，你可以通过多种方式分析数据：

1. 比较变更前后第1天的留存率（首次访问后第二天返回的用户）。第1天留存率的增加可能表明你的新入职过程更具吸引力。

2. 查看更长期的留存情况（例如，第7天、第14天、第30天），以了解改进是否有持久影响。用户在入职变更后是否留存更长时间？

3. 分析不同用户群体的留存模式，以了解新的入职过程对特定用户的有效性。

4. 如果你看到留存率改善，考虑进行A/B测试，进一步调整入职流程以不断优化该过程。

通过这种方式使用留存报告可以帮助你量化入职改进的影响，并指导进一步的优化工作。

### 用例示例：评估功能发布

在推出新功能时，你需要了解初始采用情况以及这些功能对长期用户参与的影响。留存报告可以提供帮助。

#### 设置

1. 确定新功能的发布日期。

2. 创建发布前一个月、发布月份和发布后一个月的留存报告。

3. 比较这三个时间段的留存模式。

#### 分析方法

在这些报告就位后，你可以：

1. 查找功能发布后的总体留存率变化。留存率的增加可能表明新功能驱动了更频繁的使用。

2. 分析发布后留存率的变化速度。迅速改善可能暗示用户对新功能的高度兴奋。与营销跟进，看看他们在社交渠道上是否听到了积极的反馈，或者与客户支持沟通，了解是否有任何积极反响等。

3. 考察长期留存趋势（例如，发布后2-3个月），以查看新功能是否对用户参与产生持久影响。

4. 根据用户是否使用新功能进行用户分段。比较使用和未使用该功能的用户之间的留存率，以更好地了解其影响。

5. 利用这些见解来指导未来的功能开发和资源分配。换句话说，这些数据可以帮助你确定产品路线图。

这种方法可以帮助你理解新功能对用户参与的真实影响，超越初始采用指标。

### 用例示例：季节性趋势分析

许多产品在用户行为上会经历季节性波动。理解这些模式可以帮助你更有效地规划产品更新、营销活动和资源分配。

#### 设置

1. 为每个月生成留存报告。

2. 比较不同季节或特定时期（例如假期、学期、财务季度）的留存模式。

#### 分析方法

通过这些数据，你可以：

1. 识别自然高或低留存的时期。这可以帮助你为一年中的不同时间设定更准确的留存目标。

2. 查找每年的持续模式。在夏季是否看到留存率下降，或在假日季节是否出现激增？

3. 分析不同群体在这些季节性变化中的行为。在高留存时期加入的用户是否更有可能持续留存？

4. 利用这些见解规划产品更新或营销活动。例如，你可能会在通常低留存时期专注于重新吸引用户。

5. 将你的季节性留存模式与整体使用指标进行比较，以了解季节性如何影响你的产品。

这种分析可以帮助你预测和规划用户行为的季节性变化，使你能够全年优化产品策略。

## 将留存见解与其他 Umami 报告结合

从留存报告中获得的数据可以作为起点，而其他 Umami 报告可以帮助你深入分析数据。以下是将留存报告与其他 Umami 报告结合的几种方法：

**留存 + 漏斗报告**：使用漏斗报告识别高影响用户行为，例如购买或完成教程，然后使用留存报告查看完成这些行为的用户是否具有更高的长期留存率。

**留存 + 目标报告**：在目标报告中设置基于留存的目标（例如，“X% 的用户在七天内返回”），并随时间使用留存报告跟踪这些目标的进展。

**留存 + 旅程报告**：旅程报告让你分析 3 到 7 步的用户路径，提供有关用户如何导航你的产品的见解。使用此报告识别留存率高的用户最常见路径，然后使用留存报告分析优化这些路径是否提高了其他用户群体的长期参与。

**留存 + 旅程报告以发现功能**：使用旅程报告了解用户如何发现和使用关键功能。然后，通过留存报告查看发现和使用这些功能的用户是否具有更高的留存率。这个组合可以帮助你优先考虑哪些功能需要推广或优化以改善长期参与。

利用多种 Umami 报告可以让你了解产品的表现和用户行为模式。旅程报告的复杂用户路径可视化能力与留存报告对长期参与的关注相辅相成，使你能够将特定用户行为与留存结果联系起来。

Umami 留存报告可以帮助寻求深入了解用户长期参与的产品分析团队通过可视化群体模式实现目标，它使团队能够：

1. 衡量产品变更和功能发布对短期和长期参与的影响。

2. 识别季节性趋势。

3. 设置数据驱动的留存目标并跟踪进展。

4. 优化入职和其他关键用户旅程。

5. 补充来自旅程、漏斗和目标等其他报告的见解，以全面了解用户行为。

这些当然只是一些关键示例，希望能激发你的思考。由于留存报告是基于日期的，你可以将日期与公司在产品、营销、销售、UI/UX、设计等方面进行的任何活动匹配。

审查你的留存报告和其他 Umami 报告可以提供持续的见解，以推动产品改进和提高用户参与。

Umami 的优点在于它可以与现有的分析工具一起运行，并充当第二数据源，帮助你确认从其他工具中看到的内容。
