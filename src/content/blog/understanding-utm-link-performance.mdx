如果您在链接中使用 UTM 参数，可以使用 Umami 的 UTM 报告。Umami 的跟踪脚本会自动检测并捕捉来自流量的 UTM 参数。当用户点击 UTM 链接并访问您的网站或在您的应用中浏览时，Umami：

1. 自动提取 URL 中的 UTM 参数。
2. 提取并分类每个参数（来源、媒介、活动、关键词、内容）。
3. 使这些数据在 UTM 报告中可用，无需额外设置。

这种自动提取确保您的所有 UTM 链接都被跟踪和分析，而无需额外的努力，从而使获取您工作的见解变得更加容易。

将 UTM 参数添加到链接末尾有助于您了解自己的努力，并回答“这有效吗？”这个问题。从营销到产品再到开发，UTM 可以在多种方式中使用。例如，您可以确定哪些活动、营销渠道、内容、广告、产品内部链接和知识库文章会产生效果。这些只是一些例子；将 UTM 添加到链接的用例无穷无尽。

# UTM 报告的关键特性

Umami UTM 报告的独特之处在于：

1. **UTM 参数跟踪**：报告分析所有五个标准 UTM 参数（来源、媒介、活动、关键词和内容），提供全面的活动努力视图。
2. **基于视图的分析**：通过关注视图，报告清晰展示每个活动和渠道推动的流量。
3. **灵活的日期范围选择**：选择您的分析日期范围，允许您随时间评估活动绩效并比较不同时期。
4. **自动 UTM 参数检测**：报告自动检测并分类 UTM 参数，确保所有标记的链接被跟踪和分析，无需额外设置。

让我们探讨不同团队如何利用 Umami UTM 报告来推动结果。

# 用例：功能采用跟踪

了解营销工作如何转化为功能采用对产品团队至关重要。以下是 UTM 报告如何提供帮助的简要介绍：

场景：您的团队刚刚在软件中推出了一项新功能。

1. 为宣传该功能的活动创建独特的 UTM 参数（如果您不知道如何生成 UTM 链接，谷歌有一个免费的工具）。
2. 使用 UTM 报告跟踪这些活动的流量：

   - 监控每个 UTM 组合的查看次数
   - 分析哪些电子邮件主题行或内容类型驱动最多流量

3. 将这些数据与功能使用指标交叉引用：

   - 使用 Umami 的 Journey 或 Insights 报告跟踪用户在点击 UTM 链接后路径
   - 测量通过 UTM 链接访问的用户与未访问用户的激活率

这种方法使产品团队能够：

- 确定哪些信息与用户的共鸣最好（例如，“提高生产力”与“简化沟通”）
- 了解哪些用户群体对功能公告反应最积极
- 根据成功采用该功能的用户路径优化引导流程
- 理解新功能哪些方面最吸引用户，从而为未来的功能开发提供信息

# 用例：多渠道活动评估

对于营销团队，UTM 报告是评估跨渠道活动性能的信息宝库。

场景：您正在进行一个季节性促销的多渠道活动。

1. 为每个营销渠道设置独特的 UTM 参数。例如：

   - utm_source=facebook, Instagram, Twitter, LinkedIn
   - utm_medium=social, email, paid_search, display
   - utm_campaign=fall_sale_2024
   - utm_content=carousel_ad, story_ad, promotional_email, search_ad

2. 使用 UTM 报告比较性能：

   - 分析每个渠道和内容类型的查看次数和参与率
   - 比较不同社交平台的表现
   - 评估付费流量与自然流量的有效性

3. 深入挖掘数据：

   - 使用 Umami 的 Funnel 报告跟踪来自不同 UTM 来源的转化率
   - 分析 Journey 报告以了解点击后的用户行为

对营销团队的好处包括：

- 确定最具成本效益的转化渠道
- 在活动期间实时优化广告支出
- 根据参与数据调整不同平台的内容策略
- 用数据支持的见解来通知未来的活动策略

# 用例：A/B 测试分析

开发人员可以利用 UTM 报告分析不同版本的功能或页面的有效性。

场景：您的团队正在测试新用户仪表板的两种不同布局。

1. 为每个版本创建独特的 UTM 参数：

   - utm_source=internal
   - utm_medium=ab_test
   - utm_campaign=dashboard_layout
   - utm_content=version_a, version_b

2. 使用 UTM 报告跟踪参与度：

   - 监控每个版本的查看次数和停留时间
   - 分析每个变体的跳出率和退出页面

3. 将 UTM 数据与其他指标结合：

   - 使用 Umami 的 Events 跟踪测量特定交互（例如，点击关键仪表板元素）
   - 将其与通过应用内调查收集的用户反馈交叉引用

这种方法使开发团队能够：

- 量化不同布局对用户参与度和任务完成的影响
- 根据实际用户行为做出基于数据的决策，决定推出哪个设计
- 识别每个版本中的潜在可用性问题或混淆元素
- 通过理解哪些元素驱动期望的用户行为，更有效地迭代设计

## 用例：内容策略优化

除了活动评估，营销团队还可以利用 UTM 报告来完善其内容策略。

场景：您的内容团队正在尝试不同的内容格式和主题，以用于公司的博客。

1. 为不同内容类型指定独特的 UTM 参数。例如：

   - utm_source=blog
   - utm_medium=organic_social, newsletter, internal_link
   - utm_campaign=content_strategy_2024
   - utm_content=how_to_guide, case_study, industry_report, video_tutorial

2. 使用 UTM 报告进行深入分析：

   - 比较不同内容格式和主题的参与率
   - 分析哪个分发渠道对每种内容类型最好
   - 评估常青内容与时效性内容的表现

3. 与其他数据交叉引用：
   - 使用 Umami 的 Retention 报告查看哪些内容类型驱动重复访问
   - 分析 Journey 报告，以了解用户在不同内容之间的导航

这种策略使营销团队能够：

- 确定在客户旅程不同阶段最有效的内容格式
- 了解哪些主题和主题最能引起不同受众群体的共鸣
- 通过关注高性能格式和主题优化内容创作努力
- 根据每种内容类型的渠道性能调整内容分发策略
- 制定数据驱动的编辑日历，以平衡用户兴趣和业务目标

# 访问和运行 UTM 报告

UTM 报告向所有 Umami 用户开放，您可以在 Umami 仪表盘的报告部分找到它。运行 UTM 报告非常简单：

1. 登录到您的 Umami 仪表盘
2. 导航至报告部分
3. 选择“UTM 报告”
4. 选择您希望分析的日期范围
5. 点击“运行报告”

就这么简单！您只需几次点击即可查看您的 UTM 数据。运行 UTM 报告的简单性意味着团队可以轻松将其纳入常规工作流程。无论您是在快速检查当前活动，还是在深入分析自己的营销策略，Umami UTM 报告都只需几次点击即可获取。
