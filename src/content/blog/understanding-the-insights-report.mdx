当你需要进入“研究模式”，深入了解与您网站或应用程序互动的用户的谁、什么、何时、何地、为何和如何时，洞察报告是一个很好的起点。Umami的洞察报告允许团队以几乎任何方式对数据进行细分，从而帮助发现见解，这些见解反过来又可以推动产品策略和改进。

洞察报告因其灵活性和深度而脱颖而出，帮助您发现用户与您的产品或网站互动的细微差别。无论您是希望优化用户流程、识别高价值用户群体，还是验证产品假设，洞察报告都提供了您所需的详细数据，以便做出明智的决定。

它足够灵活，可以帮助营销团队分析营销网站和他们的活动；产品团队可以分析实际应用程序及其间的一切。

# 关键特性

让我们回顾一下报告的特性：

1. **灵活的数据细分**：使用各种字段对数据进行细分，包括URL、页面标题、引荐来源、查询参数、操作系统、设备、国家、地区、城市和主机。
2. **多种指标聚焦**：分析访客、浏览量、跳出率和访问持续时间，以全面了解用户行为。
3. **高级过滤选项**：应用过滤器以关注数据的特定子集，支持“是”、“不是”、“包含”和“不包含”等操作。
4. **可定制的日期范围**：设置分析的特定日期范围，便于识别趋势和进行逐期比较。
5. **用户友好的界面**：无需技术专长即可快速创建、保存和修改报告。

# 用例示例：按流量来源分析用户参与度

了解不同流量来源如何影响用户参与度，可以帮助优化工作并改善整体用户体验。

### 设置：

1. 在字段下选择引荐来源，以按流量来源细分数据。
2. 应用过滤器以关注特定的时间段或用户群体。
3. 分析跳出率、访问持续时间和页面浏览量等指标。

### 分析方法：

1. 比较不同流量来源的参与度指标。目前，所有引荐来源都是以URL的形式存在（即，Google.com、Bing.com、facebook.com等），您将看不到“自然搜索”、“付费”、“社交”等。
2. 确定哪些来源带来最多的参与用户。
3. 检查来自不同来源的参与度在不同用户群体中的变化。

### 可操作的见解：

1. 将营销资源分配给最有效的流量来源，或修订表现不佳的流量来源的策略。
2. 根据流量来源自定义登录页面或用户体验。
3. 制定提升表现不佳来源的参与度的策略。

# 用例示例：按URL结构分析内容表现

根据URL结构分析内容表现可以揭示哪些类型的内容最能引起观众共鸣。例如，洞察报告可以帮助您分析整个博客或文档部分的汇总数据。

### 设置：

1. 从字段下拉菜单选择URL，以按内容类型或类别细分数据。
2. 使用“包含”或“以...开头”的操作应用过滤器，以分组类似内容。例如，如果您所有的博客文章的URL中都有“blog”一词，则可以使用“URL包含blog”。
3. 分析页面浏览量、平均页面停留时间和跳出率等指标。

### 分析方法：

1. 比较不同内容类别的表现指标。
2. 确定表现最佳的内容类型或主题。
3. 分析不同用户群体或设备的内容表现差异。

### 可操作的见解：

1. 将内容创作工作的重点放在最具吸引力的话题或格式上。
2. 根据成功页面的见解优化表现不佳的内容。
3. 为不同的用户群体或设备量身定制内容策略。

# 用例示例：按地理位置分析表现

了解您的产品或网站在不同地理位置的表现可以为其他策略（如付费广告、活动等）提供有趣的数据。例如，也许您的内容在德国非常受欢迎，这可能表明您的团队应测试德国市场。

### 设置：

1. 在字段下拉菜单中使用国家、地区和城市来按位置细分数据。
2. 应用过滤器以关注特定的地理区域，或比较多个地点。
3. 分析不同地区的用户参与度、转换率和功能采用率等指标。
4. 使用URL过滤器了解哪些类型的内容或单个内容在不同地区受到欢迎。

### 分析方法：

1. 比较不同国家、地区或城市的关键表现指标。
2. 确定表现优秀和表现不佳的地理区域。
3. 分析用户行为和偏好因位置而异的情况。
4. 检查地理位置与设备使用或流量来源等其他因素之间的相关性。

### 可操作的见解：

1. 制定面向特定地区的营销策略或产品功能。
2. 优先考虑高潜力市场的本地化工作。
3. 根据地区偏好优化内容或用户体验。
4. 确定并解决可能特定于某些位置的表现问题。
5. 考虑在该地点赞助一场行业活动。

# 用例示例：跨设备和平台的表现优化

我们生活在一个多设备的世界中，在所有平台上确保一致和优化的体验是一项挑战，但这是预期的。洞察报告可以帮助您识别和解决特定于某些设备或操作系统的性能问题。

### 设置：

1. 通过选择字段下的设备类型和操作系统来细分数据。
2. 使用跳出率和访问持续时间作为性能分析的关键指标。
3. 应用过滤器，关注对产品成功至关重要的特定页面或功能。

### 分析方法：

1. 比较不同设备和操作系统的跳出率及访问持续时间。
2. 确定不同平台之间性能指标的显著差异。
3. 分析不同设备上的用户路径和功能使用模式。
4. 检查性能指标与转换率或其他关键业务指标的相关性。

### 可操作的见解：

1. 优先优化表现不佳设备或操作系统的工作。
2. 确定可能需要重新设计以便更好地跨平台兼容的功能或页面。
3. 制定平台特定的策略，以在必要时改善用户体验。
4. 利用见解来指导响应式设计决策，确保在所有设备上的一致体验。
5. 调整付费广告活动。例如，如果移动设备表现不佳，可能应减少在移动设备上的预算。

通过利用洞察报告的这些过滤和细分功能，您可以深入分析用户在产品或网站不同维度上的行为。
