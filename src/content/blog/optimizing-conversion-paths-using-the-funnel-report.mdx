了解转化路径和用户旅程是优化它们的第一步。如果您已经定义了这些路径，Umami 漏斗报告（产品分析报告系列中的一部分）就适合您。漏斗报告允许您分析用户在定义过程每一步的转化和流失率，提供可以用来改善用户体验、转化率、产品特性采用率等的数据。

如果您还没有定义这些路径，且不确定用户完成某个操作所采取的路径，用户旅程报告（漏斗报告的补充报告）可能是一个合适的起点。这里不深入探讨旅程报告，但您只需定义起点、终点和中间步骤的数量，报告就会显示用户旅程。如果需要，您可以调整步骤数量并重新运行报告，以扩大您的视野。该报告提供用户交互的更广阔视图，帮助您理解用户在达到转化点之前的多种路径。

然而，本文侧重于漏斗报告，该报告允许团队可视化并分析用户如何通过特定的动作或页面序列进行移动。它识别用户流失的地方，从而揭示优化的机会。将这些数据与其他 Umami 报告的见解结合起来，您可以更清晰地了解您的表现和用户行为。

让我们开始吧。

## 漏斗报告关键特性

要有效使用 Umami 漏斗报告，您需要提前在您的产品或网站上设置自定义事件。自定义事件是您定义的特定用户行为，例如“产品添加到购物车”、“输入付款信息”、“注册新闻通讯”等。Umami 会跟踪这些事件，这些事件可以用于创建无尽具体情况的漏斗报告。在我们开始具体用例之前，先来看一下漏斗报告的一些关键特性：

1. **灵活的步骤定义**: 使用 URL、事件或 URL 通配符定义特定的漏斗步骤。

2. **时间**: 设置用户必须完成漏斗步骤的时间。这包括日期范围（最长可达 90 天）和时间窗口（用户在漏斗步骤之间必须在几分钟内完成才能计为转化）。

3. **有序步骤完成**: 分析用户通过特定顺序的步骤进行的进程。

4. **详细的流失分析**: 显示达到每一步的用户数量和从上一步的流失率。它还通过可视化的进度条和完成百分比提供即时分析。

## 漏斗报告用例

本节列出了一些漏斗报告的用例，以激发您的想法。然而，使用漏斗报告发掘用户和产品的见解几乎是无穷无尽的。
用例示例：分析注册到激活过程

产品团队需要了解新用户如何从初始注册进展为活跃用户。以下是您可以为此目的使用漏斗报告的方法：

### 设置

1. 定义您注册到激活过程中的步骤。报告至少需要两个步骤。我已经测试过最多 10 个步骤。如果您有超过十个步骤，请给我们发邮件告诉我们您的用例。这听起来真的很有趣！

2. 创建一个包含这些步骤的漏斗报告，使用 URL 或事件来表示每个阶段。

3. 设置适当的时间窗口，让用户完成漏斗。

### 分析方法

一旦您设置了漏斗报告，您可以通过以下几种方式分析数据：

1. 确定您流失最多用户的阶段，从而指示潜在的改进领域。

2. 比较用户细分或获取渠道之间的转化率，以了解哪些来源提供了最有价值的用户。

3. 分析注册或入门过程的变更如何影响转换率。

4. 利用这些见解来优先考虑产品改进或用户教育工作。

使用漏斗报告可以帮助您优化注册和激活过程，提高用户保留率和产品采用率。

## 用例示例：电商结账优化

对于专注于提高电商转化率的团队，漏斗报告可以用于理解和优化结账过程。以下是您可以采取的方式：

### 设置

1. 定义您结账过程中的阶段（例如，添加到购物车、输入送货信息、输入付款信息、确认订单）。

2. 创建一个包含这些阶段作为步骤的漏斗报告。

3. 设置适当的时间窗口，以适应典型的结账持续时间。

### 示例实施

假设您正在分析一个电商网站的结账过程。我们可以设置以下漏斗报告：

- 步骤 1: “添加到购物车”按钮点击事件
- 步骤 2: 结账页面 URL
- 步骤 3: 送货信息页面 URL
- 步骤 4: 付款信息页面 URL
- 步骤 5: “订单确认”按钮点击事件

### 分析方法

在此漏斗报告的基础上，我们可以：

1. 确定具有最高流失率的结账阶段，突出需要立即改进的领域。

2. 比较不同产品类别、用户细分或设备之间的转化率，以了解不同的用户行为。

3. 分析对结账过程的更改（例如，添加游客结账选项）如何影响整体转化率。

4. 利用见解做出关于结账页面设计、表单简化或在关键阶段添加保证元素的基于数据的决策。

此分析可以拆分结账漏斗的流向，为优化转化率提供蓝图。值得注意的是，您可以非常详细地设置此过程用于单个产品，或者在您的漏斗报告中使用通配符查看产品类别或所有产品。

## 用例示例：功能采用漏斗

对于希望增加特定功能采用的团队，漏斗报告可以提供有关用户旅程朝向功能参与的见解。以下是您可以如何使用它的方式：

### 设置

1. 确定功能采用的关键步骤（例如，功能发现、初始交互、重复使用）。

2. 创建事件或使用表示这些步骤的 URL。

3. 设置一个包含这些步骤的漏斗报告，并根据您产品的使用模式选择适当的时间窗口。

### 分析方法

假设您在一家 SaaS 公司工作，希望增加新功能的采用。您可以使用漏斗报告来：

1. 分析发现该功能的用户数量（例如，通过点击产品导览按钮等，您可能需要为不同的发现入口点设置多个漏斗）并首次使用它。

2. 确定首次使用与重复参与之间的流失，突出潜在的可用性问题。

3. 比较用户细分或入门路径之间的采用率，以了解成功采用的因素。

4. 利用这些见解完善功能引入策略、改善产品内指导或针对特定用户群体提供额外支持或鼓励。

这种方式可以帮助您根据数据做出决策，推动功能开发、优先制定产品路线图、用户教育和产品设计，鼓励采用和参与。

Umami 漏斗报告可以通过允许您可视化和分析用户在您的产品中具体路径来增强您的产品分析能力。理解这些转化漏斗将使您能够识别瓶颈、优化关键流程，并改善用户体验和转化率。

这里概述的用例只是使用漏斗报告来为您的团队提供见解的众多方式之一。其灵活性使其能够适应产品和需求的细微差别，确保其满足您的分析需求。
