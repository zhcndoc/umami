像许多其他 Google 产品一样，Google Analytics 长期以来一直是网站分析的默认选择。然而，关于数据隐私、数据所有权的担忧以及[其他问题](https://umami.is/blog/umami-the-better-google-analytics-alternative)使许多人开始寻求开源替代方案。

这时出现了 Umami。Umami 是一个简单、易用的开源网站分析工具，作为 Google Analytics 的替代品。其目标是提供一个更加友好、注重隐私的替代品，以及一个免费、开源的解决方案来替代付费产品。Umami 只收集网站所有者关心的指标，并尊重用户的隐私。

## Github 明星

Umami 的整个代码库在 [GitHub](https://github.com/umami-software/umami) 上可用，透明地展示了该工具如何收集和处理数据。这种开放性建立了信任，并促进了协作开发模型。来自世界各地的开发者为 Umami 做出贡献，建议改进，修复错误，添加新功能。

截至本文撰写时，Umami 在 GitHub 上获得了超过 22K 的星标，使其成为最流行和增长最快的开源分析项目之一。

GitHub 是 Umami 开发的中心枢纽，用户可以在这里找到最新版本，报告问题，并参与有关该工具未来方向的讨论。然而，Umami 的网站上托管有其 [文档](https://umami.is/docs)。

作为一个开源项目，任何人都可以自行审计代码库。这对于具有严格安全要求的个人和组织，或在受监管的行业中运营的企业，都是很重要的。

Umami 的开源特性允许广泛的自定义。具有特定需求的用户可以叉出仓库并修改代码，以满足他们独特的要求。这种灵活性对那些发现现成解决方案在某些方面不足的企业而言非常重要。

Umami 是免费的自托管，用户对自己的数据拥有完全控制权，这解决了与第三方分析服务（如 Google Analytics）相关的数据所有权问题。自托管选项适合那些具备技术知识和资源来管理自己基础设施的人，因为它提供了最大的控制权和潜在的成本节约。

对于没有技术背景或资源来管理自己自托管版本的用户，提供了云选项。云选项有一个免费层，允许你进行测试，还有一个慷慨的付费层，方便用户扩展。

## Umami 的功能

除了所有的 [标准分析跟踪](https://umami.is/features) 功能外，Umami 还提供 [自定义事件跟踪](https://umami.is/docs/track-events)。此功能允许网站所有者跟踪特定用户交互，例如按钮点击、表单注册、购买等。有关实施自定义事件跟踪的指南可在 Umami 的事件跟踪文档中找到。对于进一步的阅读，我们最近撰写了一篇博客文章，比较 GA4 和 Umami 的自定义事件功能：[理解自定义事件 - Umami vs. Google Analytics 4](https://umami.is/blog/understanding-custom-events-umami-vs-ga4)

[隐私](https://umami.is/blog/why-privacy-matters) 是 Umami 理念的核心。Umami 不使用 Cookies，也不收集个人信息，确保符合 [GDPR](https://umami.is/blog/gdpr-compliant-website-analytics) 和其他隐私法规，从而消除了 Cookie 同意弹窗的需求，简化了对 [数据保护法律](https://umami.is/blog/navigating-gdpr-and-ccpa-compliance) 的合规性。

Umami 最好的特点之一是其可自定义报告：

- 漏斗报告帮助分析用户路径并优化转换率。
- 留存报告可视化用户回访率和参与度随时间的变化。
- 目标报告允许设定和跟踪关键目标的进展。
- UTM 报告测量营销活动的有效性。
- 深度报告提供高级数据分段功能。
- 旅程报告帮助可视化和优化用户在网站上的旅程。

由于这篇文章是关于 Umami 的开源特性，我们不会深入详细介绍这些报告。然而，我们有丰富的资源供您探索并了解更多关于它们的信息。我们为每个报告提供了 [文档](https://umami.is/docs/reports) 和关于每个报告的博客文章，详细探讨了报告和使用案例。

Umami 是一个注重隐私的开源替代品，旨在提供 GA4（Google Analytics）分析，同时优先考虑用户隐私和数据所有权。其开源特性提供了透明度、自定义选项和社区驱动的开发。
