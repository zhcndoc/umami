在本次 Umami 版本发布中，我们引入了批量处理、负载覆盖、拒绝跟踪功能等新特性！

## 功能

### 批量发送

现在你可以批量发送数据，例如通过 API 导入大量数据时。你可以发送一个对象数组，而不是单一的负载对象。数据将通过 `/api/send` 的相同逻辑进行处理。要使用此功能，你可以发起一个 `POST` 请求到 `/api/batch`。

### 负载覆盖

当你向 `/api/send` 发送请求时，Umami 会从请求中提取诸如用户代理和 IP 地址等数据。现在你可以在负载中提供自定义值。当你从服务器或移动环境发送数据且不使用网页浏览器时，这非常有用。可以提供的值包括：

- **ip**: IP 地址。注意，当你发送自定义 IP 地址时，常见主机（如 Vercel 和 Cloudflare）提供的地理位置信息头将被忽略，系统会使用本地查询来获取国家和城市信息。  
- **userAgent**: 用户代理字符串。请确保格式有效，否则你的请求可能会被标记为机器人。  
- **timestamp**: 以秒为单位的 UNIX 时间戳。传入时，Umami 会使用该时间戳记录数据，而非当前时间。

### 拒绝跟踪 (Do Not Track)

你可以通过在你的追踪脚本中添加参数 `data-do-not-track=true` 来尊重用户的拒绝跟踪设置。
